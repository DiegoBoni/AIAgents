---
description: Turn a feature spec into an implementation plan.
arguments: Path to spec file
output: specs/<feature>/plan.md
---

## User Input

```text
$ARGUMENTS
```

## Steps

1. Read the spec and extract constraints.
2. Define technical context, architecture decisions, and milestones.
3. Add risks and mitigations.
4. Generate `plan.md` with concrete delivery phases.
5. Return plan path and critical blockers.
